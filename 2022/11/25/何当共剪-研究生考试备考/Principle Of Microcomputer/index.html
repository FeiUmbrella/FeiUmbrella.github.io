微机原理 - 大三上备考

1. 原理性内容（第二章除引脚）

8086:

  • 16根数据线+20根地址线（寻址空间：\(2^{20} = 1MB\)）

总线周期：

  • 1总线周期 = 4时钟周期
  • T1：发地址
  • T2，T3：写周期-取数；读周期-“浮空”状态
  • T4：总线周期结束
  • TW：由于取数速度较慢，故会在T3，T4之间插入若干TW来等待内存或I/O接口响应
  • TI：空闲状态-CUP不执行总线周期，总线接口部件不与总线打交道

最小模式：系统只有一个微处理器；最大模式：系统包含两个或多个微处理器（工作模式由硬件决定）

  • 复位后（reset）：从\(FFFF0H\)开始（CS：FFFFH + IP：0000H)；后面一般加一个跳转指令；复位后禁止一切可屏蔽中断。

中断：1.硬件中断（非屏蔽中断-NMI，可屏蔽中断-INTR，优先级高于软件中断）；2.软件（内部）中断。

中断子程序：

​ 与一般子程序只是返回指令不同。

​ 保护现场，开中断，处理任务，关中断，恢复现场，开中断，返回。

开中断：可以相应中断；关中断：不能相应中断。

  • 中断向量：即中断子程序入口地址；占4个字节，有256个向量，占用0段的0000~03FFH【中断向量表】。
  • 前五（类型码：0~4）个为专用中断：“除数为0中断”，“单步中断”，“非屏蔽中断（优先级最高-NMI）”，“断点中断”，“溢出中断”。
  • 由中断类型码找到中断子程序：中断类型码（17H） * 4 \(\rightarrow\) 中断子程序起始地址（5CH）\(\rightarrow\)连续4个字节的内容为中断子程序（5C~5FH）；
  • 读：读入；写：写出（CPU角度）

2.寻址方式（七种）+ 常见指令

  1. 立即数寻址：即直接给寄存器赋值

    eg. MOV AL，8

    【执行后：（AL） = 08H】

  2. 寄存器寻址：寄存器之间赋值（X：16位操作数； L/ *H：8位操作数）

    eg. MOV AX，BX

    【（AX）=3064H，（BX）=1234H；执行后：（AX）=1234H】

以下寻址方式放在代码段外的存储区中。

  • 地址 = 段地址+偏移地址

  • 基址寄存器：\(BX\) or \(BP\)

  • 变址寄存器：\(SI\) or \(DI\)

  • ==\(BP,SP\)默认段为\(SS\)；其余默认段为\(DS\)==

  1. 直接寻址方式：有效地址直接给出，默认段地址在\(DS\)段寄存器（可使用段超越前缀）

    eg.MOV AX，[2000AH]

    (DS) = 3000H，(32000H) = 3050H

    执行结果为：(AX) = 3050H

  2. 寄存器间接寻址方式：由寄存器给出有效地址。

    eg.MOV AX，[BX]

    （DS）= 2000H，（BX）=1000H，（21000H）=3050H

    执行结果为：（AX）=3050H

  3. 寄存器相对寻址：偏移量为两部分之和。 > eg.MOV AX，DISP[SI] / MOV AX，[DISP+SI] > > (DS) = 3000H，(SI) = 2000H，DISP = 3000H，(35000H) = 1234H > > （AX）=（3000H + （2000+3000）H）=（35000H）=1234H

  4. 基址变址寻址：偏移量为一个基址寄存器与一个变址寄存器之和。

    eg.MOV AX，[BX] [DI] / MOV AX，[BX+DI]

    （DS）= 2100H, (BX) = 0158H, (DI) = 10A5H, (221FDH) = 1234H

    (AX) = (2100H + (0158 + 10A5)H ) = (221FDH) = 1234H

  5. 相对基址变址寻址：偏移量为三部分之和。 > eg.MOV AX，MASK[BX] [SI] / MOV AX，[MASK + BX + SI] > > (AX) = (DS + (BX + SI + MASK)) = (33250H) = 1234H

4.cpu通过IO访问外设五种方式，对比优缺点及适用范围

I/O接口是芯片；I/O端口是芯片里的寄存器；

  1. 无条件传送方式：

前提：外设必须随时就绪。传送的接口和操作十分简单（LED数码管，按钮）。

  1. 查询方式传送/条件传送：

CPU通过执行程序不断读取并测试外设状态，在外设可以交换信息的情况下（就绪）实现数据输入，输出；多个外设，按一定顺序（优先级）依次查询。

  • 特点：工作可靠，适用面宽，但传送效率低。
  1. 中断传送方式：

进行传送的中断服务程序是预先设计好的；中断请求是外设随机向CPU提出的；CPU对请求的检测是有规律的：一般是每条指令的最后一个时钟周期采样中断请求输入引脚。

特点：费时，让外设具有申请CPU服务的主动权，有更高的效率；处理复杂传送，处理随机请求；可与CPU并行工作。

  1. DMA（直接存储器存取）传送方式：

传统：外设\(\rightarrow\) CPU \(\rightarrow\) 存储器；外设\(\leftarrow\) CPU \(\leftarrow\) 存储器。（单向）

当CPU释放总线，由DMA控制器管理，从而实现：外设\(\leftrightarrow\)存储器

  1. I/O处理机方式：功能强大，但成本较高。

==传送方式的比较==

无条件传送：慢速外设需与CPU保持同步

查询传送：简单使用，效率较低

中断传送：外设主动，可与CPU并行工作，但每次传送需要大量额外时间开销

DMA传送：DMAC控制，外设直接和存储器进行数据传送，适合大量，快速数据传送

I/O处理机方式：功能强大，但成本较高
